User Manual

Daily workflow

  1. Use the dashboard to upload a battle report, covert ops screenshot, or caravan attack screenshot.
  2. Leave server ID blank to use the parser result, or enter the server manually.
  3. The default year is the current calendar year and helps when screenshots only show month and day.
  4. Review parsed records in Recent attacks, Players, Stats, and Alliances.

Uploads

  • Battle reports track attacker, defender, alliances, server, and event time.
  • Covert ops reports track the red attacker entries from the report image.
  • For covert ops uploads, optionally enter a victim alliance so ops events count against that alliance in stats.
  • Caravan attack reports track the top caravan owner as the victim and the dated battle-history entries as attackers.
  • Caravan names can be in English, Spanish, Cyrillic, or other OCR-readable scripts.
  • Non-English parsing works best when the matching Tesseract language pack is installed on the server.
  • The same image cannot be submitted twice while it is active.
  • Click a record ID to view the submitted image and parsed details.

Manual records

  • Manual records are added from Manual Add.
  • An image is required even when entering record details by hand.
  • Alliance tags come from the portion of the player name inside brackets.
  • Example: [AVL]PlayerName records alliance AVL.

Alliances

  • The Alliances page shows attack volume, attacks received, member activity, and matchups.
  • Managed Server 78 alliances display green.
  • Other alliances display red.
  • Manage the hidden Server 78 alliance list at /admin/server78-alliances?password=mod4life.

Deletes and retention

  • Deleting a record asks for the moderator password.
  • The current delete password is mod4life.
  • Deleted records are soft-deleted, so the database row and image are retained.
  • Records older than 30 days are automatically soft-deleted on app startup.
  • Review deleted records at /admin/deleted?password=mod4life.

Discord bot

  • Invite the bot to your Discord server from the Discord Developer Portal using the application's OAuth2 bot invite URL.
  • Enable Message Content Intent for the bot in the Discord Developer Portal.
  • Start the bot with DISCORD_BOT_TOKEN set to the bot token, not the application ID or public key.
  • Set ATTACK_SCANNER_DEFAULT_SERVER=78 if most scans should default to Server 78.
  • Use @LAHT ping to check whether the bot is online. It should reply with pong.

Discord scans

  • Mention the bot and attach one or more screenshots.
  • Include battle to process the image as a battle report.
  • Include ops to process the image as a covert ops report.
  • Include caravan to process the image as a caravan attack report.
  • Example: @LAHT battle server=78 with a battle screenshot attached.
  • Example: @LAHT ops victim=AVL year=2026 with a covert ops screenshot attached.
  • Example: @LAHT caravan server=78 with a caravan attack screenshot attached.
  • For ops scans, the victim alliance can be written as victim=AVL, victim_alliance=AVL, or defender_alliance=AVL.
  • If no parser type is included, the bot tries to auto-detect the screenshot type.
  • Duplicate images are rejected the same way they are in the web dashboard.

Discord queries

  • Use @LAHT stats to show top attackers, top attacking alliances, and most attacked alliances.
  • Use @LAHT recent limit=10 to show recent records.
  • Use @LAHT history PlayerName to show a player's recent attack history.
  • Use @LAHT scanhelp to show Discord image scan options.
  • Add server=78 to filter commands to a specific server.
  • Add limit=20 to recent/history commands to control how many rows are shown.
  • The dashboard alias also works: @LAHT dashboard.

Discord troubleshooting

  • If @LAHT ping does not respond, make sure the bot process is running.
  • If mention commands do not work, confirm Message Content Intent is enabled.
  • If the bot cannot reply, check channel permissions for view channel, read message history, and send messages.
  • If web records and Discord records do not match, make sure both are using the same SQLite database path.
  • For public Discord interactions webhooks, Discord requires an HTTPS interactions URL.
